# Colors

Every color token defined in `src/styles/tokens.css`. Each swatch shows the color, its
token name, the Tailwind utility class (prefix with `bg-` / `text-` / `border-`), and the
hex value. The swatch fill renders the literal hex so it always matches the token.

## Brand

The core blue identity. Use `brand-bright-blue` for primary actions; the darker and hover
shades are reserved for interaction states.

<div className="grid grid-cols-1 gap-3 sm:grid-cols-2 lg:grid-cols-3">
  {brand.map((c) => (
    <ColorSwatch key={c.name} {...c} />
  ))}
</div>

## Accent

Secondary brand hues for highlights and illustration. Not for primary actions.

<div className="grid grid-cols-1 gap-3 sm:grid-cols-2 lg:grid-cols-3">
  {accent.map((c) => (
    <ColorSwatch key={c.name} {...c} />
  ))}
</div>

## Neutrals

Text, surfaces, and borders. `neutral-800`/`900` for primary text, `neutral-500` for
secondary, `neutral-200`/`300` for borders, `neutral-50`/`100` for surfaces.

<div className="grid grid-cols-1 gap-3 sm:grid-cols-2 lg:grid-cols-3">
  {neutrals.map((c) => (
    <ColorSwatch key={c.name} {...c} />
  ))}
</div>

## Semantic

Status colors. Each pairs a bold tone with a `-light` background tone (except `info`).

  {semantic.map((c) => (
    <ColorSwatch key={c.name} {...c} />
